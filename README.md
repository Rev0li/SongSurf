# SongSurf

YouTube Music downloader with a two-tier Docker architecture: a lightweight auth proxy (**Watcher**) permanently in front of the download engine (**SongSurf**), which is started and stopped on demand.

```
Internet → [Watcher :8080] → auth → [SongSurf :8081] → yt-dlp → FFmpeg → /data/music
```

**Admin mode** — persistent library (`Artist/Album/Title.mp3`), Beets metadata validation, Plex integration.  
**Guest mode** — temporary session with quota, automatic ZIP export, TTL cleanup.

---

## Tech stack

| Layer | Technology |
|---|---|
| Auth proxy | Python 3.11 + Flask (Watcher) |
| Download engine | Python 3.11 + Flask + yt-dlp + FFmpeg |
| Frontend | SvelteKit (adapter-static, served by Flask) |
| Tag writing | Mutagen |
| Containerization | Docker Compose (multi-file overlay) |
| Design system | MyCss tokens — rose `#E8B7C4` + teal `#6EDAD3`, light/dark mode |
| Fonts | JGS (display titles) + Geist (body) |

## Quick start (local dev)

### 1. Generate secrets

```bash
make secrets
```

Interactive — sets up `.env` (config) and `.secrets` (key material, chmod 600). Never commit `.secrets`.

### 2. Start containers

```bash
make up-local
```

App available at `http://localhost:8080`  
(port taken from `WATCHER_PORT` in `.env`, default 8080)

### 3. Other useful commands

```bash
make logs          # stream all container logs
make down          # stop containers
make restart       # down → up
make shell         # bash into running songsurf container
make dev           # run Flask backend directly without Docker (port 8081)
```

## Deployment (NAS)

```bash
make up-nas        # host network mode, JWT auth required
```

Full deployment guide: [SongSurf/DEPLOY.md](SongSurf/DEPLOY.md)

### Environment variables

Copy `.env.example` → `.env`:

| Variable | Default | Description |
|---|---|---|
| `DEPLOY_TARGET` | `local` | `local` (bridge) or `nas` (host network) |
| `WATCHER_PORT` | `8080` | Public port for Watcher |
| `DEV_MODE` | `false` | `true` bypasses JWT auth — never use in prod |
| `AUTH_SERVICE_LOGIN_URL` | _(empty)_ | rev0auth login URL (Phase 3) |
| `GUEST_MAX_SONGS` | `10` | Per-session download quota (`0` = unlimited) |
| `GUEST_SESSION_TTL` | `3600` | Guest session lifetime in seconds |

Secret material lives in `.secrets` (generated by `make secrets`): `WATCHER_SECRET`, `WATCHER_PASSWORD`, `WATCHER_GUEST_PASSWORD`, `AUTH_JWT_SECRET`.

## Architecture

### Request flow

```
Browser → Watcher :8080
  ├─ validates session / JWT
  ├─ starts SongSurf container if needed
  ├─ injects X-Watcher-Token header
  └─ proxies to SongSurf :8081
       └─ yt-dlp → FFmpeg → Organizer → /data/music/Artist/Album/Title.mp3
```

### Key files

```
watcher/
  watcher.py          Auth proxy, brute-force protection, inactivity monitor

SongSurf/
  server/
    app.py            Flask routes, download queue, guest session lifecycle
    downloader.py     yt-dlp wrapper, FFmpeg conversion, progress tracking
    organizer.py      ID3 tags (Mutagen), file tree, album art
  frontend/
    src/
      routes/         SvelteKit pages (+page.svelte, +layout.svelte)
      lib/
        styles/       design-system.css, components.css, dashboard.css
        stores.js     Svelte stores (user, theme, downloadStatus)
  Dockerfile          Multi-stage: Node build → Python serve
```

### Deployment topology

```
Internet → DuckDNS → OVH VPS (Nginx + TLS) → Tailscale → NAS (Docker)
                                                              ├─ Watcher :8080
                                                              └─ SongSurf :8081
```

Local mode uses bridge networking with published ports.  
NAS mode uses `network_mode: host`.

## Security notes

- `.secrets` is chmod 600, never committed
- Brute-force protection: 5 failed attempts → 15-minute IP lockout
- Guest errors from yt-dlp are not exposed to users
- Container runs as uid 1000 (non-root)
- Port 8081 (SongSurf) must not be exposed to WAN — only Watcher (:8080) faces external traffic
